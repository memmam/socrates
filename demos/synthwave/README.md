# synthwave â€” a chiptune track renderer in Socrates

Three voices â€” a square-wave lead, a triangle bass, and a drum kit made of
one 9-bit LFSR â€” render two bars of 12/8 at 8000 Hz into 16-bit PCM. The
samples are packed into a real RIFF/WAVE file via `std.wav` (built on
the v0.7 `Bytes` little-endian pushers), and the result is `track.wav`:
3.0 seconds of
playable chiptune, committed in this directory.

The demo then proves its own output twice:

1. **Bytes-level**: the freshly built buffer must equal the committed
   `track.wav` byte for byte (`fs.read_bytes(...) == built`, structural
   `Bytes` equality), and the RIFF/fmt/data header is decoded back *out*
   of those bytes with the native LE readers
   (`read_u32le`/`read_u16le`/`read_i16le`) â€” the pushers and the readers
   check each other.
2. **Signal-level**: `fft.rfft` analyzes rendered single notes. Pitches
   are equal temperament snapped to multiples of 4 Hz, so on a
   2000-sample window (bin width 8000/2000 = 4 Hz) each intended
   fundamental lands on an exact integer bin; `std.lists.max_by` over the
   squared magnitudes must find it there, with the 3rd harmonic as
   runner-up.

Every sample comes from `+ - * /`, comparisons, and the v0.7 bitwise
operators â€” there is deliberately no `math.sin` in the signal path â€” so
the WAV is byte-identical on every machine and safe to pin.

## Run it

From the repository root:

```sh
./target/release/socrates demos/synthwave/main.soc   # prints the track sheet, rebuilds track.wav
./target/release/socrates test demos/synthwave         # golden tests
```

Then play `demos/synthwave/track.wav` in anything that plays WAV.

## v0.7 features on display

- `Bytes`: `push_u32le`/`push_u16le`/`push_i16le` build the WAV; the
  matching readers (`read_u32le`/`read_u16le`/`read_i16le`) decode it;
  `slice(..).utf8()` reads the chunk tags; structural `==` pins the
  regeneration; `fs.read_bytes`/`fs.write_bytes` move it to disk
- `fft.rfft` as a measurement instrument: dominant-bin pitch detection
  over exact-bin notes
- bitwise `& | ^ << >>`: the LFSR noise voice (taps 9,5 â€” `checks.soc`
  pins the maximal 511-sample period)
- `std.lists` (`fill` the mix buffer, `max_by` for the spectral argmax,
  `max` for the peak), `std.set` (distinct pitches per voice, in
  first-played order), `strings.Builder` (the track-sheet rows)
- struct + impl for the `Lfsr` register, first-class `fn(Float) -> Float`
  oscillators passed between modules
