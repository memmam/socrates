# plot — a function plotter in Socrates

A small charting library and three demos built on it. Three series — a
damped sine, a damped cosine, and a cubic — are sampled into `Series`
values, framed by an affine data-to-pixel transform, and rendered as an
SVG line chart with gridlines, 1/2/5 "nice" ticks, a legend, and
collision-dodged direct labels at each line's end. A second SVG draws a
hypotrochoid ("spirograph") as 75 hue-wheel strokes on a dark canvas. A
third (new for v0.7) synthesizes a two-tone signal on exact integer bins,
pushes it through `fft.rfft`, and renders the magnitude spectrum as a stem
chart. The damped sine and the spectrum are also printed as terminal
sparklines, so the demo has output even without a browser.

## Run it

From the repository root:

```
./target/release/socrates demos/plot/main.soc            # writes into demos/plot/
./target/release/socrates demos/plot/main.soc some/dir   # writes into some/dir
./target/release/socrates test demos/plot                  # golden tests
```

The generated `plot.svg`, `spirograph.svg`, and `spectrum.svg` are
committed as sample output. Before rewriting each one, the demo compares a
fresh build against the committed bytes (`fs.read_bytes`, structural
`Bytes` equality — both v0.7) and pins the verdict, so "regenerates
byte-for-byte" is a golden line, not a promise.

## Files

| File          | What it is                                                        |
|---------------|-------------------------------------------------------------------|
| [`std.svg`](../../std/svg.soc) | a tiny SVG builder: a `Doc` accumulates elements in a `strings.Builder`, `render()` builds once — promoted from this demo's own `svg.soc`, unchanged (it was already plot-agnostic) |
| `chart.soc` | series sampling, the `Frame` transform, nice ticks, the line-chart and stem-chart renderers, sparklines |
| `main.soc`  | the three series, the spirograph, the rfft spectrum, CLI glue, and the golden `//? expect:` output |
| `checks.soc`| unit-style golden tests against the modules' public API           |
| `plot.svg`, `spirograph.svg`, `spectrum.svg` | the committed sample output        |

## Determinism notes

- The two tones sit on exact integer bins of a 128-sample window (the
  phase index `k*t` is reduced `mod n` in integer arithmetic), so the rfft
  concentrates all energy in bins 6 and 17 — no leakage, no window.
- Nothing pinned is a raw libm float: peak bins are integers picked by
  `std.lists.max_by`, magnitudes go through `to_fixed(3)`, and the leakage
  claim is a Bool at a 1e-9 tolerance. SVG coordinates are rounded to two
  decimals by `svg.num`, far coarser than any libm jitter.

## Language features on display

- multi-file structure with `import`, `pub`, and impl-block methods
- closures as data: `chart.sample` takes any `fn(Float) -> Float`
- v0.7 `strings.Builder` (+ v0.8 `push_joined`): `svg.Doc` accumulates
  element strings newline-separated and concatenates once in `render()`
- v0.7 `fft.rfft` feeding a chart, with `std.lists.max_by`/`min_by`/
  `min_float`/`max_float`/`fill` doing the extent and peak math
- v0.7 `Bytes`: `fs.read_bytes` + structural equality pin that each
  committed SVG regenerates byte-for-byte
- string interpolation building every SVG element
- `Result` + the `?` operator plumbing `fs` failures to one handler
- tuples for points, `zip`/`enumerate`/`fold`/`sort_by` on lists,
  destructured right in the loop header (`for (i, s) in …`, v0.6)
- golden tests via `//? expect:` directives (`socrates test demos/plot`)
